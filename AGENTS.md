# concur

Static communications kit for the AES "Concur + Power BI" expense-analytics pilot. It is content only: standalone HTML slide decks and email mockups (plus pre-generated PDF exports). There is no application backend, database, or build system in this repo.

Contents:
- `concur-clevel-communication.html` — executive briefing slide deck
- `expense-reports-pilot.html` — pilot meeting slide deck
- `concur-pilot-announcement-email.html` / `.pdf` — pilot announcement email
- `concur-golive-email.html` / `.pdf` — go-live announcement email

## Cursor Cloud specific instructions

- This repo has no package manager, lockfile, dependencies, or build/lint/test tooling. There is nothing to install; the only runtime needed is Python 3 (preinstalled) to serve files.
- Slide decks are self-contained (inline CSS/JS). Navigate slides with the arrow keys, click the on-screen dots, or press `F` for fullscreen.
- To preview locally, serve the repo root over HTTP and open a file in the browser:
  - `python3 -m http.server 8000` then visit e.g. `http://localhost:8000/concur-clevel-communication.html`
  - Opening the `.html` files directly via `file://` also works, but serving over HTTP is preferred for consistent relative-path behavior.
- The committed `.pdf` files are manual exports of the corresponding email HTML (via browser "Print → Save as PDF"). They are not regenerated by any script, so if you edit an email's HTML, re-export its PDF manually to keep them in sync.
- The real Concur → Power BI integration lives outside this repo; these files only describe/announce it.
